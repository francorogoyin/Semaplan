GUIA MANUAL PARA HABILITAR PLAYWRIGHT CON CAPTCHA EN SEMAPLAN

Objetivo
- Dejar una sesion valida lista para que Codex/Playwright pueda entrar
  a Semaplan sin volver a chocar con el CAPTCHA en cada intento.

Contexto
- El CAPTCHA de Cloudflare Turnstile puede bloquear Chromium
  automatizado, incluso si la ventana se ve en pantalla.
- La forma practica de destrabar esto es reutilizar una sesion ya
  iniciada desde Chrome normal.

Procedimiento recomendado
1. Abrir Chrome normal.
2. Entrar a https://semaplan.com
3. Iniciar sesion manualmente con la cuenta operativa.
4. Resolver el CAPTCHA en ese navegador normal.
5. Verificar que la sesion haya quedado realmente iniciada.
6. Dejar abierta esa sesion o avisarle a Codex para que la reutilice.

Si hace falta bypass temporal en frontend
1. Abrir la consola del navegador en Semaplan.
2. Ejecutar:
   localStorage.setItem("Semaplan_QA_Sin_Captcha", "1");
3. Recargar la pagina.
4. Esto solo evita la exigencia de CAPTCHA en frontend.
5. Si Supabase Auth sigue exigiendo CAPTCHA, tambien hay que apagarlo
   temporalmente en la configuracion hospedada.

Para volver al modo normal
1. Abrir la consola del navegador.
2. Ejecutar:
   localStorage.removeItem("Semaplan_QA_Sin_Captcha");
3. Recargar la pagina.

Que falta para que quede "bien" para automatizacion
- Idealmente, guardar y reutilizar una sesion autenticada
  (`storageState`) despues de un login manual valido.
- Si el CAPTCHA sigue frenando a nivel backend, desactivarlo
  temporalmente tambien en Supabase Auth.

Notas
- No borrar Turnstile del codigo para esto.
- Conviene tratarlo como bypass reversible de QA.
- Si la sesion expira, repetir el login manual y volver a capturarla.
