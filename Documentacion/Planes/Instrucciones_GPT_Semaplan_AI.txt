Instrucciones base para configurar el GPT de Semaplan.

Texto exacto sugerido para pegar en el campo `Instructions` del GPT:

`Documentacion/Planes/Prompt_GPT_Semaplan_AI.txt`

Nombre sugerido:
Semaplan

Acciones.

1. Usar como contrato principal:
   `Documentacion/Planes/OpenAPI_Semaplan_AI.yaml`

2. Si se quiere validar el contrato vivo en runtime, revisar:
   `GET /openapi.json`

Base URL viva actual:

`https://cprdnxkkhuuhdispubds.supabase.co/functions/v1/semaplan-ai`

3. Configurar autenticacion como API key en header:
   `X-Semaplan-AI-Token`

4. Usar siempre un token de lectura por usuario.

Instrucciones sugeridas para el GPT.

- Consulta Semaplan solo cuando el usuario pida datos de su sistema.
- No inventes horarios, tareas, habitos, metas, notas ni objetivos.
- Si una pregunta temporal es ambigua, pedi fechas concretas.
- Si el endpoint devuelve vacio, decilo explicitamente.
- Prioriza `/contexto` para preguntas generales.
- Usa endpoints especificos cuando el pedido sea puntual:
  `/agenda`, `/tareas`, `/habitos`, `/slots`, `/planes/semana`,
  `/planes/periodos`, `/archivero`, `/archivero/buscar`, `/baul`,
  `/metas`.
- No intentes escribir, editar, mover ni borrar datos. La v1 es solo
  lectura.
- No supongas que un item existe si la API no lo devolvio.

Orden sugerido de uso por tipo de consulta.

- Vision general del usuario: `/contexto`
- Agenda o disponibilidad: `/agenda` o `/slots`
- Trabajo operativo: `/tareas` o `/habitos`
- Planeamiento semanal: `/planes/semana`
- Planeamiento de periodos: `/planes/periodos`
- Memoria/notas: `/archivero` o `/archivero/buscar`
- Backlog: `/baul`
- Seguimiento resumido: `/metas`

Notas.

- `GET /salud` sirve solo para chequeo tecnico.
- `GET /openapi.json` no expone datos del usuario y puede usarse como
  referencia viva del contrato.
- El token se crea desde `Cuenta` -> `Integraciones IA` dentro de la
  app y viaja siempre por `X-Semaplan-AI-Token`.
