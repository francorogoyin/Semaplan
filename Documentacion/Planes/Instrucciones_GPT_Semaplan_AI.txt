Instrucciones base para configurar Semaplan en ChatGPT.

Modo recomendado actual: App MCP (no GPT por link).

Texto exacto sugerido para pegar en el campo `Instructions` del GPT:

`Documentacion/Planes/Prompt_GPT_Semaplan_AI.txt`

Nombre sugerido:
Semaplan

Base URL viva actual (gateway de lectura):

`https://cprdnxkkhuuhdispubds.supabase.co/functions/v1/semaplan-ai`

URL MCP remota para ChatGPT Apps:

`https://cprdnxkkhuuhdispubds.supabase.co/functions/v1/semaplan-ai-mcp/mcp`

Acciones.

1. Usar como contrato principal:
   `Documentacion/Planes/OpenAPI_Semaplan_AI.yaml`

2. Si se quiere validar el contrato vivo en runtime, revisar:
   `GET /openapi.json`

Autenticacion recomendada (sin copiar API key por usuario).

Usar OAuth en App MCP o GPT Actions.

Valores:

- Authorization URL:
  `https://cprdnxkkhuuhdispubds.supabase.co/functions/v1/semaplan-ai/oauth/authorize`
- Token URL:
  `https://cprdnxkkhuuhdispubds.supabase.co/functions/v1/semaplan-ai/oauth/token`
- Scope:
  `read`
- Client ID:
  `semaplan-chatgpt` (o el que definas en secreto)
- Client Secret:
  el mismo valor que guardes en `SEMAPLAN_AI_OAUTH_CLIENT_SECRET`

Secrets requeridos en Supabase Functions.

- `SEMAPLAN_AI_OAUTH_CLIENT_ID`
- `SEMAPLAN_AI_OAUTH_CLIENT_SECRET`
- `SEMAPLAN_AI_OAUTH_LOGIN_URL`
  recomendado:
  `https://semaplan.com/login.html`

Flujo final para usuario (App MCP).

1. En ChatGPT, abrir `Settings -> Apps & Connectors -> Create`.
2. Cargar la URL MCP remota de Semaplan (`.../mcp`).
3. Elegir autenticacion OAuth y usar URLs/credenciales de arriba.
4. Al conectar, ChatGPT redirige a Semaplan (`/oauth/authorize`) y
   luego a `login.html`.
5. Si el usuario no tenia sesion, inicia sesion en Semaplan.
6. Semaplan pide confirmar permisos de lectura.
7. Se completa OAuth y la app queda disponible para chats comunes.

Compatibilidad / fallback.

- En ChatGPT Apps (MCP), usar OAuth para identidad de usuario.
- En ChatGPT Actions, seguir usando OAuth (unico esquema del OpenAPI).
- El modo API key por `X-Semaplan-AI-Token` se mantiene para pruebas o
  scripts legacy fuera de Actions.
- En modo OAuth, cada autorizacion crea un token de lectura interno del
  usuario para auditar uso y revocar desde `Cuenta -> Integraciones IA`.

Instrucciones sugeridas para el GPT.

- Consulta Semaplan solo cuando el usuario pida datos de su sistema.
- No inventes horarios, tareas, habitos, metas, notas ni objetivos.
- Si una pregunta temporal es ambigua, pedi fechas concretas.
- Si el endpoint devuelve vacio, decilo explicitamente.
- Prioriza `/contexto` para preguntas generales.
- Usa endpoints especificos cuando el pedido sea puntual:
  `/agenda`, `/tareas`, `/habitos`, `/slots`, `/planes/semana`,
  `/planes/periodos`, `/archivero`, `/archivero/buscar`, `/baul`,
  `/metas`.
- No intentes escribir, editar, mover ni borrar datos. La v1 es solo
  lectura.
- No supongas que un item existe si la API no lo devolvio.

Orden sugerido de uso por tipo de consulta.

- Vision general del usuario: `/contexto`
- Agenda o disponibilidad: `/agenda` o `/slots`
- Trabajo operativo: `/tareas` o `/habitos`
- Planeamiento semanal: `/planes/semana`
- Planeamiento de periodos: `/planes/periodos`
- Memoria/notas: `/archivero` o `/archivero/buscar`
- Backlog: `/baul`
- Seguimiento resumido: `/metas`

Notas.

- `GET /salud` sirve solo para chequeo tecnico.
- `GET /openapi.json` no expone datos del usuario y puede usarse como
  referencia viva del contrato.
